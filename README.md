# Groovy — Language Website

> **Interested in the compiler?** This repo is the interactive website only.
> The full Groovy compiler — parser, analyzer, optimizer, code generator, and MIDI preprocessor — lives in the main project repository: **[github.com/jdillon96/Groovy](https://github.com/jdillon96/Groovy)**

---

## What Is This?

This is the official interactive website for **Groovy**, an esoteric programming language where music _is_ the source code. MIDI files serve as programs: the channel a note lives on determines what kind of character it encodes, and the pitch selects the specific character. Export a MIDI performance, run it through the Groovy compiler, and your music becomes executable JavaScript.

The website lets you explore the language without touching a terminal. You can listen to 20 real Groovy programs played as MIDI, read their source code, see the JavaScript they compile to, and write your own programs in the browser with live type-checking and in-browser compilation.

---

## Features

### MIDI Playground

The centerpiece of the site. A dropdown lets you select any of the 20 example programs. each one is a real `.mid` file encoded from actual Groovy source code. Selecting an example gives you:

- A **MIDI player** with a progress bar and animated equalizer. Click play to hear the program as music
- A **code viewer** showing the Groovy source and the compiled JavaScript side by side, switchable with a tab

### 20 Built-in Examples

Every example ships as both a `.mid` file (the playable audio) and its Groovy source, with the compiled JavaScript pre-generated by the Groovy compiler. The examples cover the full language:

| Example                | What it shows                                                 |
| ---------------------- | ------------------------------------------------------------- |
| `hello`                | Hello World — the `play` statement                            |
| `vars`                 | `note` (mutable) and `key` (immutable) variable declarations  |
| `arithmetic`           | Exponentiation (`^`), `sharp`/`flat` (`++`/`--`), expressions |
| `conditionals`         | `cue` / `alt` / `drop` (if / else-if / else)                  |
| `arrays`               | Array literals, index access, iteration                       |
| `collection`           | `measure...in` collection loop                                |
| `functions`            | `compose` function declarations with typed parameters         |
| `range`                | `measure...from...to` numeric range loop                      |
| `repeat`               | `encore N` repeat loop (unrolled by the optimizer)            |
| `while`                | `vamp` while loop                                             |
| `break`                | `cut` statement inside a loop                                 |
| `fibonacci`            | Fibonacci sequence using a `vamp` loop inside a function      |
| `gcd`                  | Euclid's GCD algorithm                                        |
| `bubble`               | Bubble sort with nested loops and array mutation              |
| `struct`               | `chord` struct declarations, constructors, and member access  |
| `optional`             | `ghost` optional types and `??` unwrap-else                   |
| `optimization`         | Constant folding, dead code elimination, loop unrolling       |
| `bachToBasics`         | A short musical program iterating over beats                  |
| `808sAndNoHeartbreaks` | A chord struct with a while loop and break                    |
| `groovyMouth`          | Functions, booleans, nested conditionals, and string return   |

### Add Your Own — Play & Compile

The **Add your own** button (next to the dropdown) opens a full in-browser editor. Write any valid Groovy program and click **Play & Compile**:

1. The code is validated against the Groovy grammar and type-checked by the full semantic analyzer
2. If valid, the compiler pipeline runs in the browser (parse → analyze → optimize → generate) and produces JavaScript output
3. `textToMidi.js` encodes your source text into a binary MIDI track using `midi-writer-js` (the same channel/pitch encoding the Groovy compiler uses), then parses it into a playable object and starts the player
4. A code block appears showing your Groovy source and the compiled JavaScript side by side

If your code has a syntax error or type error, the error message is shown in a red box instead.

## Tech Stack

| Technology                                               | Role                                                                                            |
| -------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| [React 19](https://react.dev) + [Vite](https://vite.dev) | UI framework and build tool                                                                     |
| [Tone.js](https://tonejs.github.io)                      | Web Audio API wrapper for MIDI playback                                                         |
| [@tonejs/midi](https://github.com/Tonejs/Midi)           | MIDI file parsing and playback object construction                                              |
| [midi-writer-js](https://github.com/grimmdude/MidiWriterJS) | Builds the binary MIDI track from Groovy source in the "Add your own" editor               |
| [ohm-js](https://ohmjs.org)                              | Grammar engine — powers both the validator and the in-browser compiler                          |
| Groovy compiler (`src/compiler/`)                        | The parser, analyzer, optimizer, and code generator from the main repo, adapted for the browser |

The compiler files in `src/compiler/` are copied directly from the main Groovy repo. The only browser-specific adaptation is `parser.js`, which loads `groovy.ohm` via Vite's `?raw` import instead of `node:fs`.

## Project Structure

```
src/
  components/
    Navbar.jsx          Navigation bar
    Hero.jsx            Landing hero with animated equalizer
    Description.jsx     Project description section
    LearnGroovy.jsx     Language overview and syntax guide
    Examples.jsx        MIDI Playground — player, select, editor, Play & Compile
    HighlightedCode.jsx Groovy syntax highlighter
    Credits.jsx         Team credits
  compiler/
    compiler.js         Pipeline entry point (parse → analyze → optimize → generate)
    parser.js           Browser-adapted parser (loads grammar via ?raw)
    analyzer.js         Semantic analysis and type checking
    optimizer.js        Constant folding, dead code elimination, loop unrolling
    generator.js        JavaScript code generation
    core.js             AST node factory functions
    preProcessing.js    Channel/pitch dictionaries (GROOVY_CHANNELS, REVERSE_MAP)
  data/
    data.js             The 20 examples: title, description, groovy source, compiled JS
  textToMidi.js         Encodes Groovy source into a binary MIDI track (midi-writer-js),
                        then parses it into a @tonejs/midi object for playback
  validator.js          Checks Groovy syntax using ohm-js (used for the editor error box)
  groovy.ohm            The canonical Groovy grammar
public/
  midi/                 The 20 .mid files served as static assets
```

## Running Locally

```bash
cd Diegocuadros1.github.io
npm install
npm run dev
```

To build for production:

```bash
npm run build
```

To deploy to GitHub Pages:

```bash
npm run deploy
```
